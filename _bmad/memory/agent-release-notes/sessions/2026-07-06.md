## What Happened

Born. Sanctum scaffolded via `init-sanctum.py`. Greeted Ayush, walked through `docs/` to pre-load context (existing release notes, both templates) before asking questions.

Ayush supplied a PAT for a live connectivity test. Initial `--test-connection` run failed with HTTP 401 "anonymous access" — looked like a bad PAT, but wasn't. Diagnosed by testing raw Basic auth via `Invoke-WebRequest` directly (succeeded, 200 OK), which isolated the bug to the script rather than the credential. Root cause: Windows PowerShell prepends a UTF-8 BOM when piping a string to a native process's stdin, and Python's text-mode stdin decoder on this system mangled it into 3 stray Latin-1 bytes (`\xef\xbb\xbf`) instead of one `U+FEFF` character — corrupting the Basic auth header. Fixed `scripts/fetch_work_items.py::read_pat` to read `sys.stdin.buffer` raw and decode with `utf-8-sig`, which strips the BOM correctly. Re-ran `--test-connection` — 200 OK, got the full work item type list for TWEHR.

Confirmed via conversation: owner is Ayush Pandya; releases are scoped by explicit work item ID lists (not iteration/tag sweeps); Known Issues are identified via a saved query (specifics not yet captured); there is no tag/field for Breaking Change — it's always a judgement call; template choice (internal/customer/both) depends on release and product, no fixed default.
